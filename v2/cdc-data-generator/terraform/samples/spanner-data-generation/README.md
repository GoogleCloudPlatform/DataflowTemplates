## Sample Scenario: Spanner Data Generation Job Launch

This Terraform example illustrates launching a data generation job targeting an existing Cloud Spanner database, setting up the required auxiliary staging bucket and deploying the CDC Data Generator Flex Template.

## Terraform permissions

In order to create the resources in this sample, the `Service account`/`User account` being used to run Terraform should have the required permissions. You can configure them in one of two ways:

### Using custom role and granular permissions (recommended)
Following permissions are required:
- `dataflow.jobs.create`
- `dataflow.jobs.cancel`
- `dataflow.jobs.get`
- `storage.buckets.create`
- `storage.buckets.get`
- `storage.buckets.update`
- `storage.objects.create`
- `storage.objects.get`
- `storage.objects.delete`
- `serviceusage.services.enable`
- `serviceusage.services.use`
- `iam.serviceAccounts.actAs`

> **_Note on IAM:_** If `var.common_params.add_policies_to_service_account` is set to **true** (default), the `resourcemanager.projects.setIamPolicy` permission is also required to automatically assign execution roles to your Dataflow Runner.

### Using pre-defined roles
Alternatively, you can assign the following standard GCP roles:
- `roles/dataflow.admin`
- `roles/storage.admin`
- `roles/serviceusage.serviceUsageAdmin`
- `roles/iam.serviceAccountUser`
- `roles/resourcemanager.projectIamAdmin` (required only if `add_policies_to_service_account` is set to `true`)
- `roles/viewer`

## Assumptions

This module assumes:
1. The target Spanner instance and database exist.
2. Appropriate IAM permissions are configured.

## Description

This sample contains the following files:

1. `main.tf` - Contains the Terraform resources to be created (GCS bucket, local schema/options file uploads, IAM permissions, and the Dataflow Flex Template Job).
2. `outputs.tf` - Declares the output values generated by the module (such as resource IDs and direct Google Cloud Console URLs).
3. `variables.tf` - Declares the structured input variables required to configure the resources.
4. `terraform.tf` - Declares the required providers and GCP API configurations.
5. `terraform.tfvars` - Contains the default/sample configurations to help you quickly get started.
6. `terraform_simple.tfvars` - Contains the minimal list of sample inputs to quickly deploy the module.
7. `spanner-sink-options.json` - A sample local configuration file illustrating the target project, instance, and database.

## Resources Created

This module provisions the following resources:

1. **GCS Bucket** - A bucket to hold staged configurations (sink options, schema overrides) and the default Dead-Letter Queue (DLQ) directory.
2. **Dataflow Flex Template Job** - Launches the `Cdc_Data_Generator` pipeline pointing to the container specification.
3. **Permissions** - Automatically assigns the required roles (`roles/storage.objectAdmin`, `roles/dataflow.worker`, `roles/spanner.databaseUser`) to your Dataflow Runner Service Account if `add_policies_to_service_account` is set to `true`.

## How to run

1. Clone or copy the sample folder locally.
2. Edit the `terraform.tfvars` or `terraform_simple.tfvars` file and replace the values with your own project, network, and target options GCS file path.
3. Run the following standard commands:

### Initialize Terraform
```shell
terraform init
```

### Plan and Apply
Validate the resources to be created (example using simple tfvars):
```shell
terraform plan --var-file=terraform_simple.tfvars
```

Deploy the infrastructure and launch the Dataflow job:
```shell
terraform apply --var-file=terraform_simple.tfvars
```

Upon completion, Terraform will print direct Google Cloud Console resource URLs to monitor the running Dataflow Job.

### Cleanup
To stop the running generator job and tear down the created staging bucket, run:
```shell
terraform destroy --var-file=terraform_simple.tfvars
```

## FAQ

### How can I monitor the data generation progress?
Direct links to the Google Cloud Console for the launched Dataflow job and GCS bucket are emitted as outputs upon a successful `terraform apply`.

### How can I control the generated QPS or enable max parallelism?
You can tune options such as `insert_qps`, `batch_size`, or configure the key-based redistribution parallelism by setting `max_parallelism` in your `*.tfvars` parameters block.

### Is the launched data generator a batch or streaming job?
The CDC Data Generator launches as a **streaming pipeline** in order to continuously synthesis and stream events to your target sink. Since it runs indefinitely, it will keep running and incurring standard GCP compute charges until you manually cancel it (e.g. by executing `terraform destroy`).
